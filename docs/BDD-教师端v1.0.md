# BDD：教师端 v1.0

> **对应 PRD**：PRD-教师端v1.0.md  
> **文档类型**：行为驱动开发（Behavior-Driven Development）  
> **覆盖范围**：导航布局、Dashboard、学生管理、评语系统、预警中心、数据录入、报告生成、审核中心、权限控制  
> **编写格式**：Gherkin 语法（Given-When-Then）

---

## Feature 0：教师端导航与布局（基础骨架）

> **现状**：教师端目前连左侧栏导航都没有，需要先补全基础布局。

```gherkin
Feature: 教师端左侧导航栏
  As a 教师
  I want 有一个清晰的左侧导航栏
  So that 我可以快速访问各个功能模块

  Scenario: 教师登录后看到导航栏
    Given 教师 "王老师" 已登录
    When 王老师进入教师端任意页面
    Then 页面左侧显示固定的导航栏
    And 导航栏高度占满整个视口
    And 导航栏宽度为 240px

  Scenario: 导航栏菜单项展示
    Given 教师 "王老师" 是班主任
    When 王老师查看左侧导航栏
    Then 导航栏显示以下菜单项：
      | 图标   | 名称       | 路径         |
      | 📊     | 班级看板   | /teacher     |
      | 👤     | 我的学生   | /t/students  |
      | 📝     | 写评语     | /t/comments  |
      | ⚠️     | 预警中心   | /t/warnings  |
      | 📥     | 数据录入   | /t/data-entry|
      | 📄     | 报告生成   | /t/reports   |
      | ✅     | 审核中心   | /t/review    |
    And 当前页面所在的菜单项高亮显示
    And 菜单项文字可点击跳转

  Scenario: 导航栏折叠/展开
    Given 教师 "王老师" 在使用教师端
    When 王老师点击导航栏底部的"收起"按钮
    Then 导航栏宽度收缩为 64px
    And 只显示图标，不显示文字
    And 鼠标悬停在图标上显示 Tooltip 提示菜单名称
    When 王老师再次点击"展开"按钮
    Then 导航栏恢复 240px 宽度
    And 图标和文字同时显示

  Scenario: 不同角色看到不同的导航项
    Given 教师 "刘老师" 是学科教师（非班主任）
    When 刘老师查看左侧导航栏
    Then 导航栏不显示 "审核中心" 菜单项
    And 导航栏不显示 "报告生成" 菜单项
    And "数据录入" 菜单项显示为 "本科目成绩录入"

  Scenario: 移动端导航栏
    Given 教师 "王老师" 在手机端访问教师端
    When 页面加载完成
    Then 导航栏默认隐藏
    And 顶部显示汉堡菜单按钮
    When 王老师点击汉堡菜单按钮
    Then 导航栏从左侧滑出
    And 占据屏幕宽度的 80%
    And 背景显示遮罩层
    When 王老师点击遮罩层
    Then 导航栏收回
    And 遮罩层消失
```

## Feature 1：我的班级 Dashboard

```gherkin
Feature: 班级看板 Dashboard

    And 年级逻辑维度均值 78

  I want 查看和管理我的学生

      | 逻辑 | 当前分数+标签颜色 |

    When 王老师点击某学生行的 "写评语" 按钮

    And 浮层包含：评语输入框 + 常用标签快捷选择
    And 浮层顶部显示学生姓名
    And 点击 "保存" 后浮层关闭，评语提交成功
    And 表格中该学生的评语计数 +1
```

## Feature 3：评语系统

```gherkin
Feature: 评语编辑器基础功能
  As a 教师
  I want 给学生写评语并关联六维标签
  So that 评语成为六维分数的数据来源

  Background:
    Given 教师 "王老师" 要给高一1班学生写评语

    Given 王老师进入 "写评语" 页面
    When 页面加载完成
    Then 左侧显示学生选择列表（王老师负责的学生）
    And 列表按班级分组：高一1班、高一2班...
    And 每个学生显示头像、姓名、班级、本学期已有评语数

  Scenario: 评语编辑器界面
    Given 王老师选择了学生 "张明"
    When 页面右侧加载评语编辑器
    Then 编辑器包含以下区域：
      | 区域 | 说明 |
      | 学生信息栏 | 张明 · 高一1班 · 学号2025001 |
      | 评语输入框 | 多行文本输入，支持200字以内 |
      | 字数统计 | 实时显示当前字数 |
      | 六维标签区 | 六个维度的标签网格 |
      | AI辅助按钮 | "AI生成评语" |
      | 模板选择 | 快捷模板下拉框 |
      | 预览效果 | 学生端看到的效果预览 |
      | 操作按钮 | 保存草稿 / 提交评语 |

  Scenario: 勾选六维标签
    Given 王老师正在给张明写评语
    When 王老师在 "逻辑" 维度勾选 "思维活跃"
    And 王老师在 "逻辑" 维度勾选 "逻辑清晰"
    And 王老师在 "德行" 维度勾选 "责任心强"
    Then 已勾选的标签显示为选中状态
    And 页面底部实时提示："该评语将为张明增加：逻辑+4分，德行+2分"
    And 同一维度下，标签可以多选

  Scenario: 标签重复提交限制
    Given 王老师本学期已给张明勾选了 "思维活跃"
    When 王老师再次尝试勾选 "思维活跃"
    Then 系统提示 "该标签本学期已存在，是否更新评语内容？"
    And 不重复增加分数
    And 如果教师确认更新，只更新评语文字，分数不变

  Scenario: 提交评语
    Given 王老师已填写评语内容
    And 王老师已勾选至少一个标签
    When 王老师点击 "提交评语"
    Then 系统保存评语到 Comment 表
    And 系统自动计算并更新张明的六维分数
    And 系统显示 "评语提交成功"
    And 学生端张明立即能看到这条评语
    And 页面自动切换到下一个未写评语的学生

  Scenario: 保存草稿
    Given 王老师正在写评语但还没写完
    When 王老师点击 "保存草稿"
    Then 评语内容保存为草稿状态
    And 草稿不触发六维分数更新
    And 系统提示 "草稿已保存"
    And 下次进入该学生时自动加载草稿

  Scenario: 评语预览
    Given 王老师填写了评语并勾选了标签
    When 王老师点击 "预览效果"
    Then 弹出一个模拟学生端的预览窗口
    And 预览窗口显示：评语文字 + 标签展示 + 分数变化提示
    And 教师可以确认效果后再提交
```
