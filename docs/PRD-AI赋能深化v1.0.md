# PRD：AI 赋能深化方案 v1.0

> **文档定位**：AI 能力从「辅助输入」向「智能分析」与「决策洞察」深化
> **对应系统**：教师端 v1.0 + 管理端驾驶舱
> **核心理念**：AI 不是替代教师判断，而是放大教师的观察力和管理层的决策力
> **设计原则**：数据驱动、即时可用、安全可控、渐进增强

### 模块1：AI 学情诊断报告

#### 2.1.1 功能定位

> **作为班主任/任课老师，我希望 AI 自动分析学生数据并生成诊断报告，以便我快速了解学生全貌，制定针对性干预方案。**

#### 2.1.2 触发场景

| 场景 | 触发方式 | 使用频率 |
|------|---------|---------|
| 查看学生详情 | 打开学生档案页时自动加载 | 每天 |
| 期末总结 | 教师主动点击"生成诊断" | 每学期 |
| 家长会前 | 班主任为重点关注学生生成 | 每学期 |

#### 2.1.3 输入数据

```typescript
interface DiagnosisInput {
  // 六维数据
  dimensions: Record<string, { score: number; change: number; breakdown: {...} }>;
  // 成绩趋势（最近5次考试）
  scores: Array<{ subject: string; score: number; rank: number; date: string }>;
  // 评语历史（本学期）
  comments: Array<{ teacher: string; content: string; tags: string[]; date: string }>;
  // 心情记录统计
  moodStats: { count: number; avgRating: number; trend: "up" | "down" | "stable" };
  // 活动记录
  activities: Array<{ name: string; category: string; points: number; date: string }>;
  // 里程碑
  milestones: Array<{ title: string; type: string; date: string }>;
  // 预警记录
  warnings: Array<{ type: string; description: string; status: string }>;
}
```

#### 2.1.4 输出格式

```
┌─────────────────────────────────────────────────────┐
│ 🤖 AI 学情诊断 · 张明                                │
├─────────────────────────────────────────────────────┤
│                                                      │
│ 📊 整体画像                                          │
│ 逻辑维度优秀（218分，班级Top3），但才情维度明显滞后    │
│ （198分，班级Bottom5），属于"理工型偏科"发展路径。    │
│ 近两个月身心维度持续下降，需关注是否存在学业压力。   │
│                                                      │
│ 🔍 关键发现                                          │
│ • 数学/物理成绩稳定前10%，但美术/音乐类活动零参与    │
│ • 心情记录近两周平均评分从4.2降至3.1                 │
│ • 3位任课老师均提到"课堂参与度下降"                  │
│                                                      │
│ 💡 建议行动（按优先级排序）                          │
│ 1. 【高】约谈了解近期状态变化原因                    │
│ 2. 【中】推荐参加一次艺术体验活动（+6分才情表现分）  │
│ 3. 【中】建议班主任关注其社交圈变化                  │
│                                                      │
│ 📈 如果执行建议，预期效果                            │
│ • 才情维度预计提升至205分（正常区间）                │
│ • 身心维度预计回升至220分以上                        │
│                                                      │
│ [生成家长沟通话术] [加入班级待办] [标记持续关注]     │
└─────────────────────────────────────────────────────┘
```

#### 2.1.5 AI 提示词设计

```
你是一位资深中学教育咨询师，拥有20年学生发展指导经验。

请根据以下学生数据生成学情诊断报告：

【学生数据】
{studentData}

【输出要求】
1. 整体画像：用1-2句话概括学生发展特点（优势+短板）
2. 关键发现：列出3-5个基于数据的具体发现，每条都要有数据支撑
3. 建议行动：按优先级排序，每条建议说明预期效果
4. 语气：客观、温暖、建设性，避免负面标签
5. 长度：300-500字

【约束】
- 不输出学生真实姓名（用"该生"或"该同学"）
- 不涉及心理测评原始答案
- 不使用贬低性语言
```

#### 2.1.6 技术实现

```
前端：学生详情页 → 加载时自动调用 API
            │
            ▼
API：GET /api/ai/student-diagnosis?studentId=xxx
            │
            ├── 查询学生数据（六维/成绩/评语/心情/活动/里程碑）
            ├── 检查缓存（Redis，7天有效期）
            └── 缓存未命中 → 调用 DeepSeek API 生成诊断
            │
            ▼
后端：返回诊断报告 JSON → 前端渲染
```

**缓存策略**：学生数据变化不快，诊断报告缓存 7 天。数据变更时（成绩录入、评语提交、里程碑审核）主动清除缓存。

### 模块2：AI 预警智能解读

#### 2.2.1 功能定位

> **作为班主任，我看到系统标记的预警，但不知道具体原因和该怎么办。我希望 AI 能自动分析预警背后的根因，并给出可操作的干预建议。**

#### 2.2.2 当前问题

当前预警中心只显示：
```
🟠 偏科预警 · 李华
触发：才情维度 165分（理论满分72%），其他维度均85%+
```

教师看到后一脸茫然："所以呢？我该做什么？"

#### 2.2.3 AI 增强后的预警处理流程

```
系统标记预警
    │
    ▼
AI 自动分析根因
    │
    ├── 分析该学生才情维度的历史变化趋势
    ├── 检查是否有才情相关的活动参与记录
    ├── 检查教师评语中是否从未出现才情标签
    ├── 对比班级/年级才情维度平均水平
    └── 生成根因分析和干预建议
    │
    ▼
教师看到：
┌─────────────────────────────────────────────────────┐
│ 🟠 偏科预警 · 李华                                  │
│                                                      │
│ 【AI分析】                                           │
│ 李华的才情维度（165分）低于理论满分72%，属于         │
│ "理工型偏科"。根因可能是：                          │
│ • 零艺术/文体类活动参与记录                          │
│ • 所有评语中未出现才情相关标签                       │
│ • 班级才情维度整体偏低（可能缺乏氛围）               │
│                                                      │
│ 【推荐干预】                                         │
│ ① 推荐参加艺术节（+6分）    [一键生成提醒]          │
│ ② 推荐加入美术社团（+4分）   [一键生成提醒]         │
│ ③ 课堂中多给予创意表达机会    [加入待办]            │
│                                                      │
│ [已面谈] [转介心理老师] [家校沟通] [持续关注]        │
└─────────────────────────────────────────────────────┘
```

#### 2.2.4 各预警类型的 AI 解读策略

| 预警类型 | AI 分析维度 | 干预建议来源 |
|----------|------------|-------------|
| **心理预警** | 心情记录趋势、测评历史、评语中情绪关键词 | 心理老师知识库 |
| **偏科预警** | 维度历史、活动参与、教师标签覆盖 | 六维维度-活动映射表 |
| **衰减预警** | 变化速度、同期班级趋势、重大事件 | 时间序列分析 |
| **纪律预警** | 考勤模式、违纪频率、同学关系 | 班主任经验库 |
| **低参与预警** | 活动参与率、心情记录频率、互评结果 | 参与度指标库 |

### 模块3：AI 评语质量增强

#### 2.3.1 当前状态

AI 评语助手已实现：关键词 → 生成评语草稿

#### 2.3.2 增强方向

**方向1：AI 标签自动推荐**
```
教师输入评语："张明在物理实验中表现出很强的动手能力"
    │
    ▼
AI 分析关键词 "动手能力" → 建议勾选 "创新" 维度 "动手能力强" 标签
    │
    ▼
教师点击"采纳建议" → 自动勾选
```

**方向2：AI 评语质量质检**
```
教师批量生成 25 条评语 → 点击"AI质检"
    │
    ▼
AI 逐条分析：
• 评语1：✅ 具体描述+建设性建议
• 评语2：⚠️ "表现良好"为套话，建议补充具体事例
• 评语3：❌ 包含负面标签"懒散"，建议改为"需要加强时间管理"
    │
    ▼
一键优化：AI 为问题评语生成改进版本
```

**方向3：AI 评语一致性检查**
```
检查：同一学生的多条评语是否矛盾
例：
• 数学老师："逻辑思维非常强"
• 语文老师："表达能力有待提高"
→ AI 提示：逻辑和表达维度评价一致，无矛盾 ✅

例：
• 期中评语："学习态度积极"
• 期末评语："学习态度消极"
→ AI 提示：评价存在明显矛盾，建议核实 ⚠️
```

### 4.4 内容安全过滤

```typescript
// lib/ai-safety.ts

const SENSITIVE_PATTERNS = [
  { pattern: /自杀|自残|跳楼|割腕/, level: "CRITICAL", category: "自伤" },
  { pattern: /殴打|欺负|霸凌|威胁/, level: "HIGH", category: "暴力" },
  { pattern: /废物|垃圾|没用|蠢|笨/, level: "HIGH", category: "贬低" },
  { pattern: /精神病|神经病|变态/, level: "MEDIUM", category: "歧视" },
];

export function filterAiOutput(text: string): {
  safe: boolean;
  issues?: Array<{ category: string; level: string; match: string }>;
} {
  const issues = [];
  for (const { pattern, level, category } of SENSITIVE_PATTERNS) {
    const match = text.match(pattern);
    if (match) {
      issues.push({ category, level, match: match[0] });
    }
  }
  return issues.length === 0 ? { safe: true } : { safe: false, issues };
}

// 所有 AI 输出必须经过此过滤器
export async function safeAiCall(
  prompt: string,
  generator: () => Promise<string>
): Promise<{ text: string; filtered: boolean; issues?: unknown[] }> {
  const raw = await generator();
  const check = filterAiOutput(raw);

  if (!check.safe) {
    console.error("AI 输出包含敏感内容:", check.issues);
    // 重新生成或返回安全提示
    return {
      text: "内容生成异常，请稍后重试或联系管理员。",
      filtered: true,
      issues: check.issues,
    };
  }

  return { text: raw, filtered: false };
}
```
