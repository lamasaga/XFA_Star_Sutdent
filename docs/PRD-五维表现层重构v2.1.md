# PRD：五维表现层重构 v2.1

> **文档状态**：设计稿  
> **目标版本**：v2.1  
> **核心变更**：五维从「状态导向」重构为「表现导向」  
> **新五维**：学术 · 文艺 · 体育 · 社交 · 创新  
> **关联文档**：PRD-五维雷达图数学算法重构.md（v2.0 算法基础）

---

## 一、重构背景：为什么必须改

### 1.1 现行五维的问题

| 问题 | 具体表现 | 影响 |
|------|----------|------|
| **职业维度在学生阶段格格不入** | K1-K8 学生几乎无法产生有意义的职业数据 | 职业维度 80% 时间为虚值 |
| **心理与社交重叠** | 良好社交促进心理健康，心理稳定又提升社交表现 | 同一个行为重复计分 |
| **状态维度难以被记录** | 心理/职业需要测评才能获得，学生无感知 | 反馈延迟，难以形成闭环 |
| **教师负担重** | 心理维度需要心理老师专业评估 | 普通教师无法参与 |
| **家长看不懂** | "心理78分" vs "孩子参加了3次活动" | 后者更容易理解 |

### 1.2 核心洞察：表现 vs 状态

```
┌─────────────────────────────────────────────────────────────┐
│                    现行设计（v2.0）                          │
│                                                             │
│   五维 = 学业(状态) + 心理(状态) + 职业(状态) + 社交(表现) + 特长(表现)  │
│            ↑ 成绩需要考试        ↑ 测评才知道    ↑ 可直接记录        │
│                                                             │
│   问题：3个状态 + 2个表现 = 混合架构，逻辑不一致              │
└─────────────────────────────────────────────────────────────┘

                              ↓ 重构

┌─────────────────────────────────────────────────────────────┐
│                    新设计（v2.1）                            │
│                                                             │
│   表现层（五维雷达图）                                       │
│   学术 · 文艺 · 体育 · 社交 · 创新                            │
│   ↑ 都是可被观察、记录、统计的在校表现                        │
│                                                             │
│   状态层（底层系数，不显示在雷达图）                          │
│   心理健康系数 · 职业兴趣方向                                  │
│   ↑ 影响表现层的加成/抑制，但不直接作为维度                    │
│                                                             │
│   结果：5个表现 + 2个状态 = 分层架构，逻辑清晰                │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 新五维的选取原则

每一维都必须满足：
1. **可观察**：教师/同学/家长能直接看到学生的行为或结果
2. **可记录**：不需要复杂测评，通过常规校园活动即可产生数据
3. **可比较**：不同学期、不同学生之间可以横向/纵向对比
4. **有价值**：对学生成长有意义，学校愿意投入资源支持

## 二、双层架构设计

### 2.1 架构总览

```
┌─────────────────────────────────────────────────────────────────┐
│                    用户可见层                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   🎯 五维雷达图（表现层）                                         │
│   ┌─────┬─────┬─────┬─────┬─────┐                              │
│   │学术 │文艺 │体育 │社交 │创新 │                              │
│   └─────┴─────┴─────┴─────┴─────┘                              │
│        ↑ 每个维度 0~100 = 基础分(60) + 成长分(40)                │
│                                                                 │
│   🏆 竞赛加速器（荣誉加成）                                       │
│   学术竞赛获奖 → 学术额外 +3~5                                    │
│   文艺比赛获奖 → 文艺额外 +3~5                                    │
│   体育竞赛获奖 → 体育额外 +3~5                                    │
│   科创竞赛获奖 → 创新额外 +3~5                                    │
│   辩论/演讲赛   → 社交额外 +2~4                                   │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│                    系统计算层                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   🧠 心理健康系数（0.9 ~ 1.05）                                  │
│   ┌─────────────────────────────────────┐                       │
│   │ 心理健康良好  coefficient = 1.05     │ ← 所有维度 +5%      │
│   │ 心理健康正常  coefficient = 1.00     │ ← 无影响            │
│   │ 心理健康需关注 coefficient = 0.90    │ ← 所有维度 -10%     │
│   └─────────────────────────────────────┘                       │
│        ↑ 系数每学期根据心理测评 + 心情记录计算                    │
│                                                                 │
│   🔍 职业兴趣推荐引擎                                             │
│   ┌─────────────────────────────────────┐                       │
│   │ 霍兰德测评 → 识别优势维度方向         │                       │
│   │ "艺术型"   → 推荐关注文艺维度         │                       │
│   │ "研究型"   → 推荐关注学术/创新维度    │                       │
│   │ "社会型"   → 推荐关注社交维度         │                       │
│   └─────────────────────────────────────┘                       │
│        ↑ 不直接影响分数，只提供发展建议                           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 为什么这样分层

| 层级 | 功能 | 谁关心 | 更新频率 |
|------|------|--------|----------|
| **表现层（五维）** | 展示学生在校做了什么 | 学生、家长、班主任 | 实时/每日 |
| **竞赛加速器** | 认可跨维度的卓越成就 | 学生、竞赛指导老师 | 每次竞赛后 |
| **心理健康系数** | 反映心理状态对表现的影响 | 心理老师、班主任 | 每学期 |
| **职业兴趣引擎** | 引导学生发现发展方向 | 学生、职业规划老师 | 每学年 |

**关键设计决策**：
- 心理状态不消失，只是从「维度」降级为「系数」
- 职业兴趣不消失，只是从「维度」降级为「推荐引擎」
- 学生和日常看到的雷达图更纯粹、更易懂
- 心理老师和职业规划老师仍有专业数据可用

## 三、五维表现层详细设计

### 3.0 通用规则

**显示分公式（所有维度通用）：**
```
显示分 = (基础分 + 成长分) × 心理健康系数
        = (Base + Growth) × coefficient

其中：
  Base = min(60, 40 + 在校学期数 × 3.5)  // 普通生
  Base = min(60, 40 + 在校学期数 × 5.0)  // 插班生前两学期
  Growth = Σ(各行为得分)  // 0~40，各维度独立计算
  coefficient = 0.90 ~ 1.05  // 心理健康系数
```

**标签规则（基于显示分）：**

| 显示分 | 标签 | 颜色 | 含义 |
|--------|------|------|------|
| 80~100 | **优秀** | 绿色 `#16a34a` | 该维度表现突出 |
| 65~79  | **良好** | 蓝色 `#2563eb` | 该维度发展健康 |
| 50~64  | **一般** | 黄色 `#ca8a04` | 该维度有进步空间 |
| 0~49   | **需关注** | 红色 `#dc2626` | 该维度明显滞后 |

### 3.1 学术维度（Academic）

#### 3.1.1 维度定义

学术维度衡量学生的**学科学习能力、知识掌握程度、学术研究素养**。不是简单的"成绩好"，而是"会学习、爱思考、能研究"。

#### 3.1.2 成长分构成（0~40）

```
学术成长分 = 考试成绩(0~20) + 课堂参与(0~8) + 作业质量(0~8) + 研究能力(0~4)
```

**① 考试成绩（0~20）**

| 考试类型 | 权重 | 数据来源 | 计算方式 |
|----------|------|----------|----------|
| 期末考试 | 2.0 | Score 表 | 得分率 × 10 |
| 期中考试 | 1.5 | Score 表 | 得分率 × 7.5 |
| 月考/单元测 | 1.0 | Score 表 | 得分率 × 5 |
| 学科竞赛 | 1.5 | Score 表 | 竞赛得分率 × 7.5 |

```typescript
function calculateExamScore(scores: Score[]): number {
  const semesterScores = scores.filter(s => s.semester === currentSemester);
  
  // 按科目分组，每科取最新一次
  const bySubject: Record<string, Score> = {};
  for (const s of semesterScores) {
    if (!bySubject[s.subject] || s.examDate > bySubject[s.subject].examDate) {
      bySubject[s.subject] = s;
    }
  }
  
  let weightedSum = 0;
  let totalWeight = 0;
  
  for (const score of Object.values(bySubject)) {
    const rate = score.score / (score.total || 100);
    const weight = score.examType === "FINAL" ? 2.0
                 : score.examType === "MIDTERM" ? 1.5
                 : score.examType === "COMPETITION" ? 1.5
                 : 1.0;
    weightedSum += rate * weight;
    totalWeight += weight;
  }
  
  const avgRate = totalWeight > 0 ? weightedSum / totalWeight : 0;
  return Math.min(20, Math.round(avgRate * 20));  // 封顶20
}
```

**② 课堂参与（0~8）**

| 行为 | 分值 | 记录方式 |
|------|------|----------|
| 教师评语"课堂积极"标签 | +2/条 | Comment.dimensions 中的「课堂」标签 |
| 课堂主动发言/提问 | +1/次 | 教师课堂记录（可选） |
| 小组讨论中担任汇报人 | +2/次 | 教师课堂记录（可选） |
| 提出有深度的问题 | +2/次 | 教师课堂记录（可选） |

封顶 8 分。

> **简化版（V1）**：仅通过教师评语标签计算，不增加教师记录负担。

**③ 作业质量（0~8）**

| 行为 | 分值 | 记录方式 |
|------|------|----------|
| 作业按时提交率 ≥90% | +4 | 系统自动统计 |
| 作业质量评价「优」 | +3 | 教师评语标签 |
| 作业质量评价「良」 | +2 | 教师评语标签 |
| 作业质量评价「中」 | +1 | 教师评语标签 |
| 补交/拖欠作业 | -2 | 系统自动检测 |

封顶 8 分。

**④ 研究能力（0~4）**

| 行为 | 分值 | 记录方式 |
|------|------|----------|
| 完成研究性学习课题 | +2 | 课题提交记录 |
| 研究报告/小论文被评优 | +2 | 教师评价 |
| 学术诚信无违规记录 | +1 | 系统默认（有违规则扣光） |

封顶 4 分。

#### 3.1.3 衰减机制

| 触发条件 | 衰减系数 | 说明 |
|----------|----------|------|
| 本学期无考试成绩 | ×0.90 | 考试是学术维度的核心数据源 |
| 连续两学期无考试成绩 | ×0.80 | 明显衰退 |

#### 3.1.4 典型场景

```
场景：小明本学期期中85分、期末90分、月考80分

考试成绩 = (0.85×1.5 + 0.90×2.0 + 0.80×1.0) / (1.5+2.0+1.0) × 20
         = (1.275 + 1.8 + 0.8) / 4.5 × 20
         = 3.875 / 4.5 × 20
         = 0.861 × 20 = 17.2 → 17

课堂参与 = 教师评语2条"课堂积极" = 4
作业质量 = 提交率95%(4) + 质量评价"优"(3) = 7
研究能力 = 无课题(0) + 诚信记录(1) = 1

学术成长分 = 17 + 4 + 7 + 1 = 29
基础分 = 60（假设已封顶）
心理健康系数 = 1.0（正常）

学术显示分 = (60 + 29) × 1.0 = 89（优秀）
```

### 3.2 文艺维度（Arts & Humanities）

#### 3.2.1 维度定义

文艺维度衡量学生的**文化艺术素养、审美能力、创作表达力**。涵盖音乐、美术、文学、戏剧、舞蹈、书法、传统文化等所有人文艺术领域。

#### 3.2.2 成长分构成（0~40）

```
文艺成长分 = 成果产出(0~16) + 创作实践(0~12) + 参与度(0~12)
```

**① 成果产出（0~16）**

| 行为 | 分值 | 记录方式 |
|------|------|----------|
| 技能考级通过（钢琴/书法/绘画等） | +4/级 | Milestone 表 |
| 参加校级以上演出/展览 | +3/次 | Activity 表 |
| 文艺类比赛获奖（校级+1, 区级+2, 市级+3, 省级+4） | +1~4 | Milestone 表 |
| 发表文学作品（校刊+2, 公开刊物+4） | +2~4 | Milestone 表 |

封顶 16 分。

**② 创作实践（0~12）**

| 行为 | 分值 | 记录方式 |
|------|------|----------|
| 原创作品（绘画/音乐/文学/设计） | +3/件 | Milestone 表 |
| 参与文艺社团活动 | +2/次 | Activity 表 |
| 参加文艺工作坊/大师课 | +2/次 | Activity 表 |

封顶 12 分。

**③ 参与度（0~12）**

| 行为 | 分值 | 记录方式 |
|------|------|----------|
| 每周文艺练习 ≥3小时 | +4 | 学生自主记录/教师确认 |
| 每周文艺练习 1~2小时 | +2 | 学生自主记录/教师确认 |
| 文艺社团出勤率 ≥80% | +4 | 社团签到记录 |
| 参与校园艺术节 | +2 | Activity 表 |

封顶 12 分。

#### 3.2.3 衰减机制

| 触发条件 | 衰减系数 | 说明 |
|----------|----------|------|
| 本学期无任何文艺活动记录 | ×0.75 | 文艺不练真的会退步 |
| 连续两学期无练习记录 | ×0.60 | 大幅衰退 |

#### 3.2.4 典型场景

```
场景：小红本学期钢琴六级通过、参加校园艺术节演出、每周练琴4小时

成果产出 = 钢琴六级通过(4) + 艺术节演出(3) = 7
创作实践 = 无原创作品(0) + 社团活动2次(4) = 4
参与度 = 每周练琴4小时(4) + 社团出勤(4) = 8

文艺成长分 = 7 + 4 + 8 = 19
基础分 = 60
心理健康系数 = 1.0

文艺显示分 = (60 + 19) × 1.0 = 79（良好）
```

### 3.3 体育维度（Sports & Wellness）

#### 3.3.1 维度定义

体育维度衡量学生的**身体素质、运动技能、竞技能力、健康习惯**。不是"跑得快不快"，而是"有没有运动习惯、身体好不好、能不能坚持"。

#### 3.3.2 成长分构成（0~40）

```
体育成长分 = 竞技成绩(0~14) + 身体素质(0~14) + 健康习惯(0~12)
```

**① 竞技成绩（0~14）**

| 行为 | 分值 | 记录方式 |
|------|------|----------|
| 运动会第1名 | +5 | Activity 表（result字段） |
| 运动会第2~3名 | +4 | Activity 表 |
| 运动会第4~6名 | +2 | Activity 表 |
| 运动会参与（未获奖） | +1 | Activity 表 |
| 体育比赛获奖（校级以上） | +3~5 | Milestone 表 |
| 代表学校参加区域/市级比赛 | +4 | Activity 表 |

封顶 14 分。

**② 身体素质（0~14）**

| 行为 | 分值 | 记录方式 |
|------|------|----------|
| 体测成绩「优秀」 | +5 | 体测数据导入 |
| 体测成绩「良好」 | +3 | 体测数据导入 |
| 体测成绩「及格」 | +1 | 体测数据导入 |

| 体育课成绩「优」 | +3 | Score 表（体育科目） |
| 体育课成绩「良」 | +2 | Score 表 |

封顶 14 分。

**③ 健康习惯（0~12）**

| 行为 | 分值 | 记录方式 |
|------|------|----------|

| 健康饮食记录 | +1 | 学生自评（可选） |

封顶 12 分。

#### 3.3.3 衰减机制

| 触发条件 | 衰减系数 | 说明 |
|----------|----------|------|
| 本学期无运动记录 | ×0.80 | 身体素质下降快 |
| 连续两学期无运动 | ×0.65 | 明显衰退 |

#### 3.3.4 典型场景

```

竞技成绩 = 运动会第2名(4) = 4
身体素质 = 体测良好(3) + 体育课成绩优(3) = 6
健康习惯 = 每周运动3次(4) + 出勤率100%(3) = 7

体育成长分 = 4 + 6 + 7 = 17
基础分 = 60
心理健康系数 = 1.0

体育显示分 = (60 + 17) × 1.0 = 77（良好）
```

### 3.4 社交维度（Social）

#### 3.4.1 维度定义

社交维度衡量学生的**人际协作能力、领导力、沟通表达力、社会责任感**。不是"朋友多不多"，而是"能不能合作、敢不敢担当、会不会表达"。

#### 3.4.2 成长分构成（0~40）

```
社交成长分 = 活动参与(0~14) + 角色担当(0~14) + 沟通表达(0~12)
```

**① 活动参与（0~14）**

| 连续两学期零活动 | ×0.65 | 明显孤立 |

```
沟通表达 = 演讲1次(3) + 评语"团队合作好"1条(2) = 5
```

### 3.5 创新维度（Innovation & Creation）

#### 3.5.1 维度定义

创新维度衡量学生的**创造力、问题解决能力、实践能力、科技素养**。不是"会不会编程"，而是"能不能想出新点子、能不能动手做出来、能不能改进现状"。

#### 3.5.2 成长分构成（0~40）

```
创新成长分 = 项目成果(0~16) + 创新竞赛(0~12) + 创新实践(0~12)
```

**① 项目成果（0~16）**

| 行为 | 分值 | 记录方式 |
|------|------|----------|
| 完成创新/研究项目 | +4/个 | Milestone 表 |
| 获得实用新型专利 | +6 | Milestone 表 |
| 获得发明专利/著作权 | +8 | Milestone 表 |
| 发表科创论文 | +4 | Milestone 表 |
| 作品被学校/社区采用 | +3 | 教师确认 |

封顶 16 分。

**② 创新竞赛（0~12）**

| 行为 | 分值 | 记录方式 |
|------|------|----------|
| 科创竞赛获奖（市级+3, 省级+4, 国家级+5） | +3~5 | Milestone 表 |
| 编程竞赛获奖 | +3~5 | Milestone 表 |
| 数学建模/商业挑战赛获奖 | +3~4 | Milestone 表 |
| 参与创客马拉松/黑客松 | +2/次 | Activity 表 |

封顶 12 分。

**③ 创新实践（0~12）**

| 行为 | 分值 | 记录方式 |
|------|------|----------|
| 使用创客空间/实验室完成作品 | +2/件 | 空间使用记录 |
| 完成编程作品/小程序 | +2/件 | 学生提交 |
| 提出改进方案并被采纳 | +3/次 | 教师确认 |
| 参与课题研究（非学术类） | +2/个 | 课题记录 |
| 自主发明/改进工具 | +3/件 | 教师确认 |

封顶 12 分。

#### 3.5.3 衰减机制

| 触发条件 | 衰减系数 | 说明 |
|----------|----------|------|
| 本学期无创新实践 | ×0.85 | 创新思维需要持续锻炼 |
| 连续两学期无创新实践 | ×0.75 | 衰退较快 |

#### 3.5.4 典型场景

```

创新竞赛 = 创客马拉松参与(2) = 2
创新实践 = 创客空间使用完成作品(2) + 编程作品1件(2) = 4

创新成长分 = 4 + 2 + 4 = 10
基础分 = 60
心理健康系数 = 1.0

创新显示分 = (60 + 10) × 1.0 = 70（良好）
```

## 四、底层状态层设计

### 4.1 心理健康系数

#### 4.1.1 为什么用系数而不是维度

| 对比 | 作为维度（旧） | 作为系数（新） |
|------|--------------|---------------|
| 学生理解 | "我心理78分" — 抽象 | "我的表现有5%加成" — 直观 |
| 教师参与 | 只有心理老师能打分 | 所有教师都能观察行为 |
| 家长理解 | "心理测评SCL90得分..." — 听不懂 | "孩子最近心情不错" — 听得懂 |
| 反馈速度 | 学期末测评才知道 | 每天心情记录实时可见 |
| 数据获取 | 需要专门测评时间 | 日常行为自然产生数据 |
| 干预门槛 | 心理老师预约制 | 班主任随时关注 |

#### 4.1.2 系数计算规则

心理健康系数基于两个数据源：

**① 标准化心理测评（权重60%）**

| 测评工具 | 适用年级 | 频率 | 分数映射 |
|----------|----------|------|----------|
| MHT（中学生心理健康量表） | K7-K11 | 每学期1次 | 原始分 → 0~60分 |
| SCL-90（症状自评量表） | K9-K11 | 每学期1次 | 原始分 → 0~60分 |
| 小学生心理健康问卷 | K1-K6 | 每学期1次 | 原始分 → 0~60分 |

**② 心情日记连续性（权重40%）**

| 心情记录情况 | 得分 | 说明 |
|-------------|------|------|
| 本学期记录 ≥30天 | 40 | 极佳 |
| 本学期记录 14~29天 | 32 | 良好 |
| 本学期记录 7~13天 | 24 | 一般 |
| 本学期记录 3~6天 | 16 | 较少 |
| 本学期记录 <3天 | 8 | 严重不足 |
| 连续30天记录 | 额外+5 | 连续性奖励 |
| 连续14天记录 | 额外+3 | 连续性奖励 |

**综合计算：**

```typescript
function calculateMentalHealthCoefficient(
  assessmentScore: number,  // 0~60
  moodScore: number          // 0~45（含连续性奖励）
): number {
  // 测评得分标准化到 0~60
  const normalizedAssessment = Math.min(60, assessmentScore);
  
  // 心情得分标准化到 0~45（封顶）
  const normalizedMood = Math.min(45, moodScore);
  
  // 加权总分（满分100）
  const totalScore = normalizedAssessment * 0.6 + (normalizedMood / 45 * 60) * 0.4;
  
  // 映射到系数 0.90 ~ 1.05
  // 50分 = 1.0（正常基线）
  if (totalScore >= 70) return 1.05;  // 心理健康良好
  if (totalScore >= 55) return 1.02;  // 心理健康较好
  if (totalScore >= 45) return 1.00;  // 心理健康正常
  if (totalScore >= 35) return 0.95;  // 心理健康一般
  return 0.90;                          // 心理健康需关注
}
```

**系数效果示例：**

```
学生A：测评65分 + 心情记录25天(32+5连续奖励)
  totalScore = 65×0.6 + (37/45×60)×0.4 = 39 + 19.7 = 58.7
  coefficient = 1.02
  
  学术显示分 = (60+28) × 1.02 = 89.8 → 90（+2分加成）

学生B：测评40分 + 心情记录5天(16)
  totalScore = 40×0.6 + (16/45×60)×0.4 = 24 + 8.5 = 32.5
  coefficient = 0.90
  
  学术显示分 = (60+28) × 0.90 = 79.2 → 79（-9分抑制）
```

#### 4.1.3 系数使用规则

| 系数值 | 等级 | 学生端显示 | 教师端显示 | 干预动作 |
|--------|------|-----------|-----------|----------|
| 1.05 | 良好 | "心理状态佳，表现有加成 💚" | 绿色标识 | 无需干预 |
| 1.02 | 较好 | "心理状态不错 💙" | 蓝色标识 | 鼓励保持 |
| 1.00 | 正常 | 不显示 | 无标识 | 正常关注 |
| 0.95 | 一般 | "最近状态一般，注意休息 💛" | 黄色标识 | 班主任关注 |
| 0.90 | 需关注 | "建议多关注自己 💔" | 红色标识 | 心理老师介入 |

**重要原则：**
- 系数只影响**显示分**，不影响**成长分**的原始计算
- 成长分反映真实努力，系数只是"放大器/缩小器"
- 学生可以看到系数提示，但系数本身不单独展示为分数
- 教师端可以看到系数及构成（测评分+心情分）

### 4.2 职业兴趣推荐引擎

#### 4.2.1 为什么从维度降级为推荐引擎

| 问题 | 说明 |
|------|------|
| K1-K6 无意义 | 小学生对职业没有概念 |
| K7-K9 太浅 | 初中生只有模糊兴趣 |
| K10-K11 才刚有概念 | 高中生开始思考，但选择有限 |
| 职业数据产生频率低 | 一年做一次测评，分数长期不变 |

**结论**：职业兴趣不是需要持续刷分的维度，而是帮助学生**发现自己可能适合什么方向**的工具。

#### 4.2.2 推荐引擎设计

```typescript
interface CareerRecommendation {
  dominantType: string;        // 主导类型，如"艺术型"
  secondaryType: string;       // 次要类型，如"研究型"
  recommendedDimensions: string[];  // 推荐关注的表现维度
  suggestedActivities: string[];    // 推荐参加的活动
  growthBonus: number;         // 推荐维度的成长分加成系数（1.0~1.1）
}

function generateRecommendation(assessment: Assessment): CareerRecommendation {
  const holland = parseHolland(assessment.resultJson);
  
  const typeToDimension: Record<string, { dims: string[]; activities: string[] }> = {
    "现实型": { dims: ["体育", "创新"], activities: ["参加科技社团", "尝试动手制作"] },
    "研究型": { dims: ["学术", "创新"], activities: ["参加学科竞赛", "做研究性课题"] },
    "艺术型": { dims: ["文艺", "创新"], activities: ["加入乐队/戏剧社", "尝试原创"] },
    "社会型": { dims: ["社交", "文艺"], activities: ["参加义工", "担任班干部"] },
    "企业型": { dims: ["社交", "创新"], activities: ["组织活动", "参加商业挑战赛"] },
    "常规型": { dims: ["学术", "文艺"], activities: ["参加学术社团", "尝试写作"] },
  };
  
  const mapping = typeToDimension[holland.primary] || { dims: ["学术"], activities: [] };
  
  return {
    dominantType: holland.primary,
    secondaryType: holland.secondary,
    recommendedDimensions: mapping.dims,
    suggestedActivities: mapping.activities,
    growthBonus: 1.05,  // 推荐维度成长分 +5% 加成（鼓励顺势而为）
  };
}
```

#### 4.2.3 推荐引擎的使用场景

**场景1：Dashboard 待办推荐**

> 💡 **基于你的霍兰德测评结果（艺术型+研究型），你在「文艺」和「学术」维度有天然优势！**
> 推荐待办：
> - [ ] 尝试创作一件原创文艺作品（文艺维度 +5% 成长加速）
> - [ ] 参加一次学科竞赛（学术维度 +5% 成长加速）

**场景2：学期档案建议**

> 你的职业兴趣方向为「社会型」，建议你本学期多关注「社交」维度的发展。可以尝试担任班级职务或参加社区服务。

**场景3：教师端参考**

教师查看学生档案时，可以看到职业兴趣推荐，据此给予个性化指导：
- "企业型"学生 → 推荐组织活动、担任项目负责人
- "研究型"学生 → 推荐参加学科竞赛、做研究课题

#### 4.2.4 推荐引擎的边界

| 做什么 | 不做什么 |
|--------|----------|
| 推荐方向和建议 | 不强制学生走某条路 |
| 给优势维度小幅加成（+5%） | 不给劣势维度惩罚 |
| 帮助学生发现可能性 | 不替代学生自己的判断 |
| 每学期更新一次 | 不频繁推送干扰学生 |

## 五、竞赛加速器

### 5.1 为什么竞赛不是独立维度

| 竞赛类型 | 本质上属于 | 如果单列维度的问题 |
|----------|-----------|------------------|
| 数学奥赛 | 学术 | 学术竞赛型学生天然多一个维度 |
| 钢琴比赛 | 文艺 | 文艺比赛型学生天然多一个维度 |
| 运动会百米 | 体育 | 体育竞赛型学生天然多一个维度 |
| 机器人大赛 | 创新 | 造成重复计分 |
| 辩论赛 | 社交 | 不公平的维度优势 |

**结论**：竞赛是「跨维度的卓越表现认证」，应该在对应维度上给予额外奖励，而不是成为第六维。

### 5.2 竞赛加速规则

| 竞赛类别 | 影响维度 | 校级 | 区级 | 市级 | 省级 | 国家级 |
|----------|---------|------|------|------|------|--------|
| **学科竞赛**（数学/物理/化学/生物/信息学） | 学术 | +2 | +3 | +4 | +5 | +6 |
| **文艺比赛**（音乐/美术/舞蹈/戏剧/书法） | 文艺 | +2 | +3 | +4 | +5 | +6 |
| **体育竞赛**（田径/球类/游泳/武术） | 体育 | +2 | +3 | +4 | +5 | +6 |
| **科创竞赛**（机器人/编程/发明/创客） | 创新 | +2 | +3 | +4 | +5 | +6 |
| **演讲/辩论赛** | 社交 | +1 | +2 | +3 | +4 | +5 |
| **综合竞赛**（全能/综合素质） | 最高维度 | +2 | +3 | +4 | +5 | +6 |

**规则说明：**
- 竞赛加分直接加在**成长分**上，但不超过40封顶
- 同一竞赛在多个维度获奖，取**最高加分**（不叠加）
- 团体竞赛中，所有队员获得**70%**加分（鼓励团队协作）
- 竞赛加分**不影响基础分**和**心理健康系数**

### 5.3 竞赛加速器示例

```
学生小明：参加市级数学竞赛获二等奖

学术成长分原始计算 = 25
竞赛加速器 = 市级学科竞赛 +4
学术成长分（加速后）= min(40, 25 + 4) = 29

学术显示分 = (60 + 29) × 1.0 = 89 → 93（+4分）

如果同时获得省级物理竞赛三等奖：
学术成长分（加速后）= min(40, 25 + 5) = 30  // 取省级加分（更高）
```

## 七、全面发展保障机制

### 7.1 问题：学生只刷一个维度怎么办？

**场景**：学生发现"只要拼命考试，学术维度就会很高"，于是只关注成绩，忽视其他方面。

### 7.2 对策

#### 对策1：均衡度评价

```typescript
function calculateBalanceScore(dimensions: Record<string, number>): number {
  const values = Object.values(dimensions);
  const mean = values.reduce((a, b) => a + b, 0) / values.length;
  const variance = values.reduce((sum, v) => sum + Math.pow(v - mean, 2), 0) / values.length;
  const stdDev = Math.sqrt(variance);
  
  // 均衡度：标准差越小越均衡
  return Math.max(0, Math.round(100 - stdDev * 2));
}
```

| 均衡度 | 评价 | 说明 |
|--------|------|------|
| 90~100 | **非常均衡** | 五维齐头并进，全面发展典范 |
| 75~89 | **较为均衡** | 有强弱之分，但整体健康 |
| 60~74 | **略有偏科** | 某个维度明显落后 |
| <60 | **偏科严重** | 需要干预和引导 |

**示例：**

```
学生A：[92, 88, 85, 90, 87]  均值=88.4, 标准差=2.1, 均衡度=96 ✅
学生B：[96, 62, 58, 65, 60]  均值=68.2, 标准差=15.8, 均衡度=68 ⚠️
```

| 创新最低 | 去一次创客空间 | 提出一个改进小建议 | +3~5 |

## 八、插班生适配

### 8.1 插班生的特殊挑战

| 挑战 | 说明 |
|------|------|
| 基础分差距 | 在校生基础分已封顶60，插班生从40起步 |
| 无历史数据 | 雷达图"从零开始"，找不到自己的位置 |
| 不了解体系 | 不知道学校有哪些社团、活动、竞赛 |
| 社交从零开始 | 没有朋友，社交维度数据为空 |
| 文艺/体育特长未被发现 | 需要重新展示自己 |

### 8.2 插班生专属机制

#### 机制1：观察期基础分加速

| 学期 | 插班生基础分 | 普通生同期基础分 | 差距 |
|------|-------------|-----------------|------|
| 入学学期 | 40 | — | — |
| 第1学期结束 | 45 | 43.5 | +1.5 |
| 第2学期结束 | 50 | 47 | +3 |
| 第3学期结束 | 53.5 | 50.5 | +3 |
| 第4学期结束 | 57 | 54 | +3 |
| 第5学期结束 | 60 | 57.5 | +2.5 |
| 第6学期起 | 60 | 60 | 持平 |

**结论**：观察期（前两学期）让插班生多获得 4.5 分的基础分优势，快速缩小差距。

#### 机制2：五维新手保护

插班生入学前两个学期，各维度成长分有**保底值**：

| 维度 | 新手保底成长分 | 触发条件 |
|------|--------------|----------|
| 学术 | 10 | 参加一次考试或提交一次作业 |
| 文艺 | 10 | 参加一次文艺活动或社团体验 |
| 体育 | 10 | 参加一次体育课或运动会 |
| 社交 | 10 | 参加一次集体活动或自我介绍 |
| 创新 | 8 | 参观一次创客空间或科技展 |

**说明**：只要完成最低参与门槛，就不会因为"还没摸清门路"而被判低分。

#### 机制3：快速发现特长

插班生入学第一学期，系统会：
- 推荐参加**所有类型**的社团体验活动（学术/文艺/体育/创新各一次）
- 记录学生在哪些活动中表现积极
- 第一个月后生成「特长初探报告」，推荐可能的优势维度

#### 机制4：社交破冰引导

```
Dashboard 插班生专属待办：

🎉 欢迎来到新府学！
本周推荐：
□ 参加新生欢迎会（社交+5）
□ 加入一个兴趣社团（对应维度+5）
□ 记录本周心情（心理系数加成）
□ 完成霍兰德测评（获得职业兴趣推荐）

💡 提示：前两个学期是观察期，基础分加速增长！
```

#### 机制5：虚拟历史回填（可选）

如插班生携带外部成绩/活动记录：
- 生成「虚拟快照」，标注 `isVirtual: true`
- 在雷达图历史趋势中用**虚线**展示
- 帮助插班生看到自己过去的成长轨迹
