# BDD：六维成长体系 v3.1

> **对应 PRD**：PRD-六维成长体系v3.1-修订.md  
> **文档类型**：行为驱动开发（Behavior-Driven Development）  
> **覆盖范围**：成长分三层结构、六维计算、标签体系、插班生机制、衰减机制、学生互评、雷达图展示、教师端操作  
> **编写格式**：Gherkin 语法（Given-When-Then）

---

## Feature 1：成长分三层结构

> **核心原则**：底线分（不违纪就有60%）+ 表现分（正常成绩+参与）+ 卓越分（竞赛+获奖+老师好评）

### Feature 1.1 底线分计算

```gherkin
Feature: 底线分计算
  As a 系统
  I want 根据学生的基础行为记录计算底线分
  So that 守规矩的学生至少能获得60%的成长分保底

  Background:
    Given 学生 "小明" 是 K8 在读学生
    And 当前学期为 "2025-2026-1"

  Scenario: 逻辑维度底线分 — 守规矩学生拿满36分
    Given 小明本学期数理课程无迟到早退记录
    And 小明本学期数理课程无缺勤记录
    And 小明本学期数理作业提交率 ≥ 70%
    When 系统计算逻辑维度底线分
    Then 逻辑底线分应为 36 分
    And 系统标记 "基础行为达标"

  Scenario: 逻辑维度底线分 — 有缺勤记录扣减
    Given 小明本学期数理课程迟到 2 次
    And 小明本学期数理课程缺勤 1 次
    And 小明本学期数理作业提交率为 60%
    When 系统计算逻辑维度底线分
    Then 逻辑底线分应为 24 分
    And 系统提示 "数理课程缺勤1次，建议关注出勤情况"

  Scenario: 身心维度底线分 — 正常参与拿满36分
    Given 小明本学期体育课无迟到早退
    And 小明本学期体育课无缺勤
    And 小明本学期完成体测（无论成绩）
    And 小明本学期记录心情日记 ≥ 3 次
    And 小明本学期完成心理测评
    When 系统计算身心维度底线分
    Then 身心底线分应为 36 分

  Scenario: 德行维度底线分 — 无违纪记录拿满36分
    Given 小明本学期无严重违纪记录
    And 小明本学期无作弊记录
    And 小明本学期无欺凌/暴力记录
    When 系统计算德行维度底线分
    Then 德行底线分应为 36 分
    And 系统显示 "品行良好"

  Scenario: 德行维度底线分 — 有违纪记录
    Given 小明本学期有 1 次严重违纪记录
    And 小明本学期无作弊记录
    And 小明本学期无欺凌/暴力记录
    When 系统计算德行维度底线分
    Then 德行底线分应为 24 分
    And 系统触发 "德行预警" 给班主任
```

### Feature 1.2 表现分计算

```gherkin
    Given 学生 "小红" 是 K9 在读学生
    And 当前学期为 "2025-2026-1"

  Scenario: 逻辑维度表现分 — 成绩中等

  Scenario: 逻辑维度表现分 — 成绩良好
    Given 小红本学期数学班级排名 30%
    And 小红本学期物理班级排名 25%
    And 小红本学期化学班级排名 35%
    When 系统计算逻辑维度表现分
    Then 逻辑表现分应为 8 分
```

### Feature 1.4 成长分汇总

```gherkin

  As a 系统
  I want 将三层分汇总为总成长分
  So that 得到完整的成长分

  Scenario: 正常学生的成长分

      | 层级   | 分数 |
      | 底线分 | 36   |
      | 表现分 | 10   |
      | 卓越分 | 2    |
    When 系统汇总逻辑成长分

    And 标签为 "良好"

  Scenario: 优秀学生的成长分
    Given 学生 "小丽" 的逻辑维度：
      | 层级   | 分数 |
      | 底线分 | 36   |
      | 表现分 | 12   |
      | 卓越分 | 12   |
    When 系统汇总逻辑成长分
    Then 逻辑成长分应为 60 分
    And 小丽达到理论满分
    And 标签为 "超前"

  Scenario: 问题学生的成长分
    Given 学生 "小强" 的逻辑维度：
      | 层级   | 分数 |
      | 底线分 | 20   |
      | 表现分 | 2    |
      | 卓越分 | 0    |
    When 系统汇总逻辑成长分
    Then 逻辑成长分应为 22 分
    And 小强显示分/满分比例约为 83%
    And 标签为 "正常"
    And 系统触发 "需关注" 预警
```

## Feature 2：六维总分计算

### Feature 2.1 基础分（年级基准分）

```gherkin
Feature: 年级基准分计算
  As a 系统
  I want 根据学生所在年级计算基准分
  So that 不同年级的学生有公平的起点

  Scenario Outline: 各年级基准分
    Given 学生所在年级为 "<grade>"
    When 系统查询年级基准分
    Then 基准分应为 <benchmark>

    Examples:
      | grade | benchmark |
      | K1    | 40        |
      | K2    | 60        |
      | K3    | 80        |
      | K4    | 100       |
      | K5    | 120       |
      | K6    | 140       |
      | K7    | 155       |
      | K8    | 170       |
      | K9    | 182       |
      | K10   | 192       |
      | K11   | 200       |

  Scenario: 插班生基础分 — 从实际入学年级开始
    Given 学生 "小张" K8 插班入学
    And 当前年级为 K8
    When 系统计算小张的基础分
    Then 基础分应为 170（K8 基准）
    And 不继承其他在校生的历史基础分

  Scenario: 插班生观察期加速
    Given 学生 "小李" K8 秋插班入学
    And 当前为 K8 春学期
    When 系统计算小李的基础分
    Then 基础分增长为每学期 +5（而非普通生的 +3.5）
    And 系统显示 "观察期加速中" 标签

  Scenario: 基础分封顶
    Given 学生 "小王" 已在读 8 个学期
    And 小王不是插班生
    When 系统计算小王的基础分
    Then 基础分达到封顶 60
    And 此后基础分不再增长
    And 显示分增长完全依赖成长分
```

### Feature 2.2 显示分计算

```gherkin

  Scenario: 六个维度全部计算
    Given 学生 "小红" 是 K9 学生
    And K9 年级基准分为 182
    And 小红的各维度成长分为：
      | 维度 | 成长分 |

      | 创新 | 45     |
      | 表达 | 50     |
      | 才情 | 38     |
      | 身心 | 48     |

    Then 各维度显示分为：

      | 创新 | 227    |
      | 表达 | 232    |
```

## Feature 4：衰减机制

```gherkin
Feature: 成长分衰减机制
  As a 系统
  I want 在学生长期不参与某维度时触发衰减
  So that 学生意识到需要持续关注每个维度

  Background:
    Given 学生 "小李" 上学期各维度成长分为：
      | 维度 | 成长分 |
      | 逻辑 | 30     |
      | 创新 | 20     |
      | 表达 | 25     |
      | 才情 | 15     |
      | 身心 | 22     |
      | 德行 | 28     |

  Scenario: 学业维度小幅衰减
    Given 小李本学期逻辑维度无任何有效数据
    When 系统计算本学期逻辑成长分
    Then 逻辑成长分 = 30 × 0.9 = 27
    And 系统提示 "本学期数理活动较少，建议保持参与"

  Scenario: 心理维度中等衰减
    Given 小李本学期身心维度无任何有效数据
    When 系统计算本学期身心成长分
    Then 身心成长分 = 22 × 0.85 = 18.7 → 19
    And 系统提示 "建议记录心情日记或参加体育活动"

  Scenario: 特长维度大幅衰减
    Given 小李本学期才情维度无任何有效数据
    When 系统计算本学期才情成长分
    Then 才情成长分 = 15 × 0.7 = 10.5 → 11
    And 系统提示 "艺术活动参与度下降明显，建议重新参与"
    And 标签从 "良好" 降为 "正常"

  Scenario: 连续两学期无数据 — 二次衰减
    Given 小李上学期才情成长分为 15
    And 小李上上学期才情成长分为 22
    And 小李已连续两学期才情维度无有效数据
    When 系统计算本学期才情成长分
    Then 才情成长分 = 22 × 0.7 × 0.7 = 10.8 → 11
    And 系统触发 "才情维度持续下降" 预警

  Scenario: 衰减最低保留5分鼓励分
    Given 小李上学期某维度成长分为 6
    And 本学期该维度无数据
    And 衰减系数为 0.7
    When 系统计算成长分
    Then 成长分 = 6 × 0.7 = 4.2 → 4
    And 系统取最低保留值 5 分
    And 系统提示 "只要重新开始参与，分数就会回升！"

  Scenario: 衰减不影响基础分
    Given 学生 "小王" 基础分已封顶为 60
    And 上学期特长成长分为 25
    And 本学期特长无任何数据
    When 系统计算特长显示分
    Then 基础分仍为 60
    And 成长分 = 25 × 0.7 = 17.5 → 18
    And 显示分 = 60 + 18 = 78（上学期为 85）
    And 系统提示 "特长维度下降 7 分，建议重新开始训练"
```

## Feature 5：插班生机制

```gherkin
Feature: 插班生快速融入机制
  As a 插班生
  I want 获得系统的特殊支持
  So that 我能快速融入新环境，不因起点低而丧失信心

  Background:
    Given 学生 "小新" K8 春学期从外校转入
    And 小新被标记为插班生

  Scenario: 插班生基础分从实际入学时间计算
    Given 小新 K8 春入学
    And 当前为 K8 春学期
    When 系统计算小新的基础分
    Then 基础分从 K8 基准 170 开始
    And 不与其他 K1 入学的学生对比历史

  Scenario: 插班生观察期加速
    Given 小新处于插班生观察期（前两个学期）
    And 当前为入学后的第 1 个学期
    When 系统计算基础分增长
    Then 每学期增长 +5（而非普通生的 +3.5）
    And 系统显示 "观察期加速" 标签

  Scenario: 插班生新手保护
    Given 小新是插班生
    And 本学期小新只完成了一次心理测评
    And 本学期小新参加了 1 次活动
    When 系统计算各维度成长分
    Then 参与过的维度成长分保底 10-15 分
    And 系统显示 "新手保护" 标签
    And 不因 "还没摸清门路" 被判低分

  Scenario: 观察期结束后恢复正常增长
    Given 小新已完成 2 个学期的观察期
    And 当前为第 3 个学期
    When 系统计算基础分增长
    Then 基础分增长恢复为每学期 +3.5
    And "观察期加速" 标签消失

  Scenario: 插班生 Dashboard 专属提示
    Given 小新登录学生端
    When 小新查看 Dashboard
    Then 显示插班生专属欢迎提示："欢迎来到新府学！这是你在这里的第 N 个学期。前两个学期是观察期，系统会给予额外鼓励分。"
    And 显示专属待办清单引导完成关键行为

  Scenario: 虚拟历史快照（携带外部记录）
    Given 小新转入时携带了公立学校的成绩记录
    When 管理员导入外部记录
    Then 系统生成 K1-K7 虚拟快照
    And 雷达图上用虚线展示过去轨迹
    And 标注 "虚拟数据（来自原学校）"
```

```gherkin

      | 标签       |

      | 友善待人   |
      | 乐于助人   |
      | 责任心强   |
      | 团队合作好 |

    Given 小明对小红勾选了 "友善待人"

    Then 小红看到 "友善待人：65% 同学勾选"

      | 标签       | 勾选比例 |
      | 值得信赖   | 70%      |
      | 友善待人   | 65%      |
      | 乐于助人   | 45%      |
      | 责任心强   | 55%      |
    When 系统计算小红的德行表现分
```
