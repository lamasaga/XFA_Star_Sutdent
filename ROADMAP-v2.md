## 四、关键设计决策建议

### 4.1 家长端：独立路由还是共享？

**建议**：独立路由 `/parent`，独立登录页。

理由：
- 家长端是只读视图，权限模型简单（只看自己孩子）
- 不占用学生端的路由空间，避免 `/profile` 变复杂
- 未来可扩展为小程序/H5，独立路由更方便

### 4.2 学期档案：实时生成还是预生成？

**建议**：预生成 + 缓存。

流程：
1. 期末教师点击"生成档案"
2. 后端异步任务聚合数据（成绩+评语+里程碑+活动）
3. 生成网页版（存入数据库）+ PDF 版（存入对象存储）
4. 学生/家长打开时直接读取，无需实时计算

### 4.3 AI 功能：通用模型还是微调？

**建议**：先用通用 DeepSeek + Prompt Engineering，暂不微调。

理由：
- 当前数据量不足以支撑有效微调
- Prompt 中加入教师历史修改记录（3~5 条）即可模仿风格
- 等积累 1000+ 条评语数据后再考虑微调

### 4.4 游戏化：积分与现有等级如何衔接？

**建议**：统一积分池，等级 = f(积分)。

```
积分来源：
- 心情打卡：+5/天，连续 7 天额外 +10
- 五维自评：+20/周
- 里程碑申报（通过）：+10~50
- 活动参与：+10~30
- 教师评语质量分：+0~20

等级公式：Lv = floor(√(积分/10))
Lv.1 → Lv.50 对应 10 → 25000 积分
```
